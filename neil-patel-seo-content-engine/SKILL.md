## The Core Framework: STEPPS Content Method

Neil Patel's content success comes down to 5 principles:

### 1. Search Intent First
Before writing a single word, ask: **What does the searcher actually want?**

The 4 types of search intent:
- **Informational**: "how to start dropshipping" (teach them)
- **Navigational**: "Shopify pricing" (direct them)
- **Commercial**: "best dropshipping suppliers" (compare options for them)
- **Transactional**: "buy dropshipping course" (convert them now)

**Rule:** Your content format must match intent. Informational = guide. Commercial = comparison. Transactional = landing page.

### 2. Topical Authority Over Individual Posts
Neil Patel doesn't write random posts. He builds topic clusters:

**Pillar Page** (2,000-5,000 words): Covers the broad topic completely
Example: "The Complete Guide to Dropshipping"

**Cluster Posts** (800-1,500 words each): Cover subtopics in depth
- "How to Find Winning Dropshipping Products"
- "Best Dropshipping Suppliers for 2026"
- "How to Write Dropshipping Product Descriptions That Sell"
- "TikTok Ads for Dropshipping: Beginner's Guide"

All cluster posts link back to the pillar. The pillar links to all clusters. Google sees the site as an authority on the topic.

### 3. The Skyscraper Technique
Find the #1 ranking article for your keyword. Make yours:
- 40% longer
- More up-to-date (add 2026 data/examples)
- More visual (more subheadings, bullets, tables)
- More actionable (add step-by-step sections, templates)

### 4. Hook + Value Density Formula
Neil Patel's blog intro structure (first 150 words):
1. **Stat hook**: Open with a surprising number
2. **Problem validation**: "Most [people] struggle with [X]"
3. **Promise**: "In this guide, you'll learn exactly how to [result]"
4. **Proof**: Quick credibility (used by X companies, X results)
5. **Transition**: "Let's dive in"

### 5. Conversion Architecture
Every post has a purpose beyond traffic:
- **Email capture**: Lead magnet offer midway through long posts
- **Next step CTA**: Link to a related tool, product, or guide
- **Exit intent**: Triggered when reader finishes content
- **Internal links**: At least 3-5 links to related content per post

## Step-by-Step Prompts

### Prompt 1: Full Blog Post Generator
```
You are an SEO content writer trained on Neil Patel's frameworks.

Write a complete, publish-ready blog post for:

Target keyword: [exact keyword you want to rank for]
Secondary keywords: [2-3 related terms]
Search intent: [informational/commercial/transactional]
Target audience: [who reads this -- beginner/intermediate/advanced, their situation]
Business goal: [drive email signups / sell product / build authority]
Word count: [800 / 1,500 / 2,500+]

Structure requirements:
- Hook intro with a surprising stat (first 100 words)
- H2 sections with H3 subsections
- At least one numbered list and one bullet list
- One comparison table if applicable
- FAQ section at the end (5 questions, for featured snippet targeting)
- Meta title (60 chars max) and meta description (155 chars max)
- CTA at the end: [describe your offer]

Write in a conversational, direct tone. No corporate language. Use "you" throughout.
```

### Prompt 2: Content Brief Generator
```
Create a detailed SEO content brief for a blog post targeting: [keyword]

Include:
1. Primary keyword + 5 LSI (semantically related) keywords
2. Search intent classification + what format this demands
3. Recommended word count based on top 3 competitors
4. H1 title (3 options)
5. Recommended H2 sections (7-10 headings)
6. Key points to cover under each H2
7. Competitor content gaps (what they're missing that I should include)
8. Recommended internal linking topics
9. Featured snippet opportunity: write a 40-60 word definition paragraph for [topic]
10. Meta title and description
```

### Prompt 3: Pillar Page + Cluster Map
```
I'm building topical authority in the [niche] space.

My main pillar topic is: [broad topic]

Create:
1. A pillar page outline (H1 + 10 H2 sections with H3 subsections listed)
2. 8 cluster article titles that support this pillar
3. For each cluster article: target keyword, search intent, recommended word count
4. Internal linking plan: which cluster links point to the pillar, and which pillar sections link out to clusters
5. The pillar page meta title and meta description
```

### Prompt 4: Product/Ecommerce SEO Post
```
Write a Neil Patel-style SEO blog post for an ecommerce/dropshipping store.

Store niche: [your product niche, e.g., pet supplies, fitness gear]
Target keyword: [e.g., "best yoga mats for beginners"]
Funnel stage: [TOFU - awareness / MOFU - consideration / BOFU - purchase ready]

Include:
- Stat-based hook intro
- Product comparison table with 5 options (columns: product, price, best for, rating)
- Our product featured as #1 or #2 with the most detailed breakdown
- "What to look for when buying [product]" section (builds trust)
- FAQ section targeting "people also ask" questions
- CTA to shop: [your store URL or product page]

Tone: Helpful expert, not salesy. Think "Consumer Reports" not "infomercial".
```

### Prompt 5: Skyscraper Content Upgrade
```
I want to outrank the current #1 result for "[keyword]".

Their article: [paste URL or describe the existing top article briefly]
Their approximate word count: [X words]
Their weak spots (gaps, outdated info, missing sections): [describe]

Write an improved version that:
- Is [X + 40%] words long
- Includes everything they have + these missing sections: [list gaps]
- Adds 2026-current statistics and examples
- Has a better intro hook
- Has a featured snippet-optimized definition in the first H2
- Ends with a stronger CTA

My business context: [your product/service/niche]
```

## Content Calendar Framework

Neil Patel publishes consistently. Use this ratio for any niche:

| Week | Content Type | Purpose |
|---|---|---|
| 1 | Pillar page (2,500+ words) | Topical authority anchor |
| 2 | Cluster post A (1,200 words) | Target long-tail keyword |
| 3 | Cluster post B (1,200 words) | Target comparison keyword |
| 4 | Cluster post C (800 words) | Target FAQ/question keyword |

Repeat monthly. After 6 months: update your pillar page with new data.
