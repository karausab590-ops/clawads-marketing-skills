# ClawAds Skill Builder

Research, write, and publish a new marketing skill to clawads.io. This skill teaches you how to build skills that any AI assistant can use - OpenClaw, Claude Code, Cursor, or any MCP-compatible tool.

## How to Use

Tell me what marketing topic, guru, or framework you want to turn into a skill. I'll research the framework, write a complete SKILL.md, and guide you through publishing it.

**Just say:** "Build a skill for [topic/guru/framework]"

---

## What Makes a Good Skill

A skill is not a prompt template. A skill is a **framework transfer** - it teaches the AI how an expert thinks, not just what to output.

### The Standard of Quality

A good skill has:
1. **Why this approach is different** - what insight separates this expert's method from generic advice
2. **The actual framework** - named steps, formulas, or decision process, explained deeply
3. **The psychological mechanism** - WHY each element works (brain response, emotion triggered)
4. **Rules and anti-patterns** - what NOT to do, common mistakes that kill results
5. **Ready-to-use output format** - copy-paste copy, complete scripts, nothing to edit

### What to Avoid
- No lists of 'types' without explaining when/why to use each
- No vague descriptions without mechanism
- No examples = abstract and unusable
- No generic output format - define the structure precisely
- No guru name without the guru's actual thinking

## The Skill Research Process

### Step 1: Identify the Unique Insight
Every great marketing skill is built around ONE insight that makes the expert's approach work.

Examples:
- **John Caples:** Test everything. Opinion is irrelevant. Response rates are facts.
- **Eugene Schwartz:** You can't create desire. You can only channel desire that already exists.
- **Gary Halbert:** The hungry crowd beats everything. Even a bad offer to a hungry crowd wins.
- **David Ogilvy:** Research before writing. The consumer is not stupid - they're your spouse.

### Step 2: Identify the Framework Components
Named elements with specific order and logic. Each does ONE specific job. Can apply to any product.

### Step 3: Find the Anti-Patterns
What does this expert specifically warn against? Anti-patterns are often more useful than positive advice.

### Step 4: Define the Output
Be specific: How many outputs? What format? What metadata per item (formula, why it works, where to use)?

## Platform Compatibility

Skills built this way work on any AI platform:

- **OpenClaw**: File SKILL.md, install via clawads.io Download ZIP
- **Claude Code / Claude Projects**: Same content saved as CLAUDE.md in project root
- **Cursor**: Paste into .cursorrules or CLAUDE.md
- **Any AI assistant**: Copy content, paste as system prompt

**Key principle:** Write skills that teach thinking, not tool commands. A skill that says 'think like John Caples' works everywhere.
